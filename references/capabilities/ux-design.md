# UX Design Capability

Use UX design guidance when flows, interaction states, information architecture, accessibility, or responsive behavior materially affect the feature.

## Fallback Guidance

Cover:

- Primary users and jobs.
- Entry points and exits.
- Screen or state flow.
- Empty, loading, error, success, and permission states.
- Keyboard and assistive technology behavior.
- Responsive layout behavior.
- Copy and validation rules.

Keep UX guidance practical and tied to the spec.
